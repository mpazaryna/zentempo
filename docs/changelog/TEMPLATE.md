# Changelog Entry Template

Use this template when adding detailed changelog entries for significant changes.

---

## [Feature/Fix/Change Title]
*[Added/Changed/Fixed/Removed]: YYYY-MM-DD*

### Overview
Brief description of what was changed and why.

### Problem (for fixes)
What issue was being experienced?

### Root Cause (for fixes)
Technical explanation of why the problem occurred.

### Solution/Implementation
- Key implementation details
- Technical decisions made
- Important code changes

### User Impact
How this affects end users:
- New capabilities
- Changed behaviors
- Required actions

### Technical Notes
- Architecture considerations
- Performance implications
- Dependencies affected
- Future considerations

### Testing
- How to verify the change
- Edge cases considered
- Known limitations

### Migration Guide (if breaking change)
Steps users need to take to adapt to this change.

### Related
- Issue: #XXX
- PR: #XXX
- Discussion: [link]

### Media
Screenshots or recordings demonstrating the change (store in `media/` subfolder).
