# Documentation

This directory contains user-facing documentation, guides, and reference materials for the Zentempo application.

## Purpose

The `docs/` folder is designed for:
- **User guides** and tutorials
- **API documentation** (if applicable)
- **Installation instructions**
- **Usage examples** and best practices
- **Troubleshooting guides**
- **Release notes** and changelogs

## Why Separate from Plans?

This separation provides clear boundaries:
- **docs/** = Stable, user-facing documentation for end users
- **plans/** = Internal planning and design documents for developers

## Structure

```
docs/
├── README.md           (this file)
├── user-guide/        (end-user documentation)
├── troubleshooting/   (common issues and solutions)
├── releases/          (version history and release notes)
└── media/            (screenshots, diagrams, etc.)
```

## For End Users

This folder contains everything needed to:
- Understand how to use Zentempo
- Troubleshoot common issues
- Learn about new features and updates

## Documentation Standards

When adding documentation:
1. Write for the end user, not developers
2. Include screenshots where helpful
3. Keep language clear and concise
4. Test all instructions before documenting
5. Update for each significant release

## Current Documentation

As Zentempo evolves, user documentation will be added here including:
- Quick start guide
- Keyboard shortcuts reference
- Settings and customization guide
- FAQ and troubleshooting

## Note for Developers

For technical planning, architecture decisions, and implementation details, see the `/plans` directory.
